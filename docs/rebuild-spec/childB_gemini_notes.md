# childB (Gemini) 補足メモ

## 1. 特殊ターン（OD／追加ターン）状態の明文化が不足
- READMEでは **5.3.1～5.3.2** でOD1～OD3や追加ターンの表記・継続・SP効果を列挙しており、**10.1** でも特殊ターン管理がv1.0要件に含まれる（`README.md:163-178, 270-276`）。
- 現在の制御ロジック（`js/control-manager.js:88-112`）では `currentTurn` を単純なインクリメントで扱い、OD／追加ターンを区別するフラグや継続ターン数、発動対象キャラのアクション制限が存在しないため、仕様記録の保存や後続処理の分岐に必要な状態遷移が不明。CSV出力でも各行に「OD1」「追加1」などのラベルを載せるルールが定まっていない。
- 要件として、特殊ターンの開始条件・継続ターン数・対象範囲・SP回復量の計算順序（通常ターンとの重複回復や前衛交代時の影響）を明示し、`battleHistory` や CSV に格納するキー名／列名も決める必要がある。

## 2. CSV/スプレッドシート出力仕様の詳細定義がない
- README では **9章** で「3行ヘッダー+データ」「E-V列に1人3列」「B列に特殊ターン表記」など細かいフォーマットを示しているが、実装では結果表示が `js/results-manager.js:1-49` にあるようにあくまで DOM テーブルの書き換えで、ファイル出力や列ラベルの構造定義、バッファ保持方式が記述されていない。
- そのため実際に CSV を生成する場合の列順・カンマ区切りの処理、特殊ターンやバフ・デバフ情報の記録場所、行内での配置履歴（6人のポジション変化）をどこで補足するのかが曖昧。
- CSV出力の責務を担うクラス/関数のインタフェース（入力：`battleHistory`＋配置変更ログ、出力：文字列／URL／バイナリ）と、各行に含める「変動要因」や「特殊ターンラベル」を定義する資料が必要。

## 3. SP加減算・パッシブ効果の積み重ねルールが未定義
- README **4章** では初期SPの装備・パッシブ展開（他者への効果、前衛条件付）や、ターン経過・前衛/後衛の追加回復、パーティー全体/特定キャラへの支援など多層的な効果を列挙している（`README.md:132-218`）。
- 実装（`js/control-manager.js:102-112`）では `BASE_SP_RECOVERY + character.spBonus` で一律に回復し、 `spBonus` はパーティー編成UIで入力できる単一の値に過ぎず、他キャラへの波及や追加ターン時の+5などの条件付き効果を表現できない。
- したがって、パッシブ／配置／追加ターン／ODごとの適用順序、重複時の合算ルール（加算順／中央値／上限など）、他キャラへのバフ配布を構造化して記述する必要がある。どのデータ要素が定義すべきか（sourceId, targetScope, triggerTiming, magnitude, cap）を明文化すると後続実装が容易になる。
