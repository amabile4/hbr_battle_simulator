## 📊 心眼・フィールドの帰属カテゴリ調査結果 (T3.R4.3)

### ケース C: 心眼 (MindEye) の帰属カテゴリ
- **Excel での計算**:
  - バフ合計セルではなく、心眼専用のセル `AJ59` で集約されます。
  - 詳細計算エリア（`BK266`等）の乗算式：`*(100% + SUM(AJ47:AJ49) + AJ52 + AJ59 + AJ34 + AJ41 + AJ45 + AX49%)`
  - つまり、Excel では **「スキル攻撃力アップカテゴリ（AttackUp 等と加算）」に属して計算されています**（クリティカルカテゴリには含まれません）。
- **Python/JS エンジンでの計算**:
  - `crit_scale = (1.5 + crit_buff_total + mindeye_buff_total) / 1.5`
  - つまり、エンジン側では **「クリティカルカテゴリ」に加算して計算されています**。
- **差異と照合結果**:
  - 両者で **帰属カテゴリに設計上の差異** があります。ただし、心眼が「クリティカル発動かつ弱点攻撃時」のみ有効であるゲーム内仕様上、どちらの実装でも最終的なクリティカルダメージの倍率は同等にスケーリングされますが、通常ダメージ時の計算処理を含めると Excel 側は AttackUp 内、エンジン側は Critical 内で処理している点が異なります。

### ケース D: フィールド (Zone/Field) の帰属カテゴリ
- **Excel での計算**:
  - フィールド専用セル `AS32` で合計値（最大80%など）が算出されます。
  - 詳細計算エリア（`BK266`等）の乗算式：`*(100% + AS32)`
  - つまり、Excel では **「独立した乗算因子」として AttackUp カテゴリとは独立して乗算されています**。
- **Python/JS エンジンでの計算**:
  - `resistance_total = affinity_mult * zone_mult` （該当ゾーン時 `zone_mult = 1.5`）として、耐性補正と乗算されます。
- **差異と照合結果**:
  - Excel・エンジン側ともに **「独立した乗算因子（乗算）」として処理されており、実装方針は完全に一致しています**。
