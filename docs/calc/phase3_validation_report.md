## 2. 検証再現性および実行環境

検証用テストコードおよびテストデータは、テスト実行環境の依存性（Excel解析モジュールや大規模データ容量など）を分離するため、`hbr_calc` リポジトリ側で一元管理されています。

### A. 前提条件と環境構築
検証を実行するには、`hbr_calc` リポジトリ内に `hbr_battle_simulator` へのシンボリックリンクを作成する必要があります。
```bash
# hbr_calc ディレクトリにて実行
ln -s ../hbr_battle_simulator/json json
ln -s ../hbr_battle_simulator/src src
```

### B. 各検証の実行手順とファイル所在

#### ① 固定境界ケース検証 (Fixed Fixtures)
- **概要**: 基礎ダメージ、バフ枠キャップ、デバフ上限分類、脆弱弱点判定、クリティカル心眼枠、連撃トークン乗算など、仕様書で定義された主要な境界値における数式モデルの検証。
- **使用スクリプト**: `hbr_calc/run_js_fixed_fixtures_tests.mjs`
- **入力データ**: `hbr_calc/test_cases_fixed.json` (6ケース)
- **実行コマンド**:
  ```bash
  node run_js_fixed_fixtures_tests.mjs
  ```

#### ② Excel格子点突合回帰テスト (Regression)
- **概要**: Excel（`HBR計算機🎭Ver.4.31.03.xlsx`）の有効な115行から抽出したパラメータと計算結果（期待通常・クリティカルダメージ等）との差分検証（許容誤差 `1e-4` 以下）。
- **使用スクリプト**: `hbr_calc/run_js_regression_tests.py` （内部で `run_js_calc_helper.mjs` を呼び出し）
- **入力データ**: `hbr_calc/HBR計算機🎭Ver.4.31.03.xlsx`
- **実行コマンド**:
  ```bash
  uv run python run_js_regression_tests.py
  ```
  *(注: macOS環境での実行権限の問題を避けるため、`uv run python` 経由での実行を推奨)*

#### ③ 大規模ランダム差分テスト (Differential) & カバレッジ測定
- **概要**: 力・器用さなどのステータス（300〜1000）、レベル、限界突破、ランダムバフ・デバフ、戦闘ゾーンなどを網羅した2,000件のランダムケースにおける、Python版とJS版の出力の同一性検証。
- **使用スクリプト**: `hbr_calc/run_js_large_tests.mjs`
- **入力データ**: `hbr_calc/test_cases_large.json`（`generate_test_cases_large.py` により生成）
- **実行コマンド**:
  ```bash
  node --experimental-test-coverage --test run_js_large_tests.mjs
  ```

## 4. 仕様変更と Excelモデルとの乖離要因

回帰テストにおいて突合を行うため、検証用スクリプト上で「Excel互換レイヤー（データモックの注入など）」を適用しています。これは、JS/TS移植版において実データ駆動への改善や仕様の適正化を行ったことに起因する正常な乖離です。

### 1) 非攻撃スキルのダメージクランプ (0.0クランプ)
- **内容**: `ソフニング` などのデバフ専用スキルについて、ExcelおよびPython版（Excel模倣時）では数式コピーの都合上ダメージが算出されていましたが、JS版では「攻撃Partを持たないスキルは基礎ダメージを0にする」という仕様が正しく適用されています。

### 2) 条件分岐スキル（OD有無・使用回数による分岐）の扱い
- **内容**: `石塔の手筋+` などのスキルについて、Excelでは「通常時」「OD時」等の条件ごとに別行として表現されていますが、JS版ではマスタの構造に則って状態の静的解決（第1要素の処理）を正しく行っています。

### 3) 手動SP / 閾値調整の適正化
- **内容**: 一部スキルにおいて、Excelモデルでは消費SPなどを手動で書き換えた上で計算していましたが、JS版は `skills.json` 内の実ゲームデータに準拠して計算を行うため、より実際のゲーム内容に則した値を出力するよう改善されています。

## 5. Category Fallback Warning の扱いについて（T3.4.1）

`damage-calculator.js` の一部処理には、`category` が未定義の場合にスキル名からカテゴリ（`DefenseDown` など）を推定するフォールバックロジックが存在しています。
- **計算結果への影響**: 本検証における 2,000 件のテストケースおよび Excel 突合検証において、このフォールバックによる計算結果の狂いは発生しておらず、**非ブロッカー（計算の正当性に影響を与えないもの）**であることを確認しています。
- **今後の対応方針**: このフォールバックは、既存 fixture との互換性維持のために残されています。より厳密な設計（strict contract）へ移行して名称ベースのフォールバックを完全に排除するかどうかについては、フェーズ4（UI・画面連携）の進捗を見ながらフォローアップタスクとして検討を継続します。
