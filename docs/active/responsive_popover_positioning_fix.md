# レスポンシブポップアップ配置修正 - WBS

## 問題概要
iPhone画面上でターンのレコードを入力中にブレイクボタンを押してブレイクするターゲット選択のポップアップを出した時、場所が固定のため、クリック可能な場所が画面外に表示される問題の調査と修正。

### 1.1 コードベース構造分析
- [x] `ui-next/` ディレクトリ構造を探索
- [x] ターン入力に関連するUIコンポーネントを特定
- [x] ポップアップ/ポップオーバー実装ファイルを特定

### 1.2 原因特定
- [x] `ui-next/components/turn-row.js` を調査
- [x] `ui-next/styles.css` を調査
- [x] ポップオーバーのCSSクラスと配置を分析
- [x] ハードコーディングされた配置値を特定

**調査結果:**
- ブレイクエディタポップオーバー: `.target-popover` に `right-0 top-[calc(100%+4px)] z-30 w-[280px]`
- ターゲット選択ポップオーバー: `.target-popover` に `right-0 top-[calc(100%+4px)] z-20 min-w-[180px]`
- 両方ともボタンの右端に固定配置
- 固定幅（ブレイクエディタ280px、ターゲットポップオーバー最小180px）
- ビューポート境界チェックなし
- 狭い画面でのレスポンシブ調整なし

**根本原因:**
狭い画面（iPhone）でブレイク/ターゲットボタンが右端付近にある場合、ポップオーバーがビューポート外にはみ出す理由:
1. `right-0` でボタンの右端にアンカー
2. 固定幅（280px）がコンテンツを画面外へ押し出す
3. 利用可能なビューポートスペースに基づいて位置を変更する仕組みがない

## 解決設計フェーズ

### 2.1 CSSレスポンシブ改善
- [x] `.target-popover` のレスポンシブCSS戦略を設計
- [x] ビューポート単位を使用した最大幅制約を定義
- [x] 狭い画面向けメディアクエリアプローチを設計
- [x] 画面幅に基づく左右アンカースイッチを計画

**CSS戦略:**
- `max-width: min(280px, calc(100vw - 16px))` を追加してオーバーフローを防止
- メディアクエリ `@media (max-width: 640px)` で左アンカーに切り替え
- 広い画面では一貫性を保つため右アンカー維持

### 2.2 JavaScript位置調整
- [x] ビューポート境界チェックロジックを設計
- [x] 動的アンカースイッチメカニズムを計画
- [x] 位置再計算のトリガーを定義
- [x] パフォーマンスの確保（過剰な計算回避）

**JavaScript戦略:**
- TurnRowControllerに `adjustPopoverPosition(popoverEl, triggerEl)` メソッドを追加
- ポップオーバー境界とビューポート幅を計算
- ポップオーバーが右端を超える場合、左アンカーに切り替え
- ポップオーバー表示時に調整を適用
- ブレイクエディタとターゲット選択ポップオーバーの両方を考慮
