# GUI開発技術候補

> **ステータス**: 📚 参照 | 📅 最終更新: 2026-03-08

本ドキュメントでは、シミュレータ等の複雑なUIを持つWebアプリケーションの調査結果から得られた、今後のGUI開発・改修において利用可能な技術スタックの候補をまとめています。特に、キャラクターアイコンのドラッグ＆ドロップなどの直感的な操作を実現するための技術に焦点を当てています。

## 1. フロントエンド（UI構築）

### [React](https://reactjs.org/)
- **概要**: コンポーネント指向のUI構築ライブラリ。
- **特徴**:
  - ドラッグ＆ドロップのような複雑な状態管理とDOM更新を効率的に行える。
  - エコシステムが非常に充実しており、目的に合ったライブラリ（今回のDnDなど）を見つけやすい。
- **採用理由（候補）**: 昨今のモダンWebアプリケーション開発におけるデファクトスタンダードであり、複雑なGUIシミュレーターの開発に最適。

### [Tailwind CSS](https://tailwindcss.com/)
- **概要**: ユーティリティファーストのCSSフレームワーク。
- **特徴**:
  - `class="col-span-6 relative flex"` のように、HTMLに直接クラスを記述して素早くスタイリングできる。
  - デザインの統一感を保ちながら、カスタムCSSを書く量を大幅に減らせる。
- **採用理由（候補）**: 開発スピードの向上と、保守性の高いスタイリングを実現するため。

## 2. ドラッグ＆ドロップ (DnD) 実装ライブラリ

React環境下で、リストアイテムの並び替えやエリア間の移動（シミュレータのキャラクター配置など）を実現するためのライブラリ候補です。

### 候補1: [@hello-pangea/dnd](https://github.com/hello-pangea/dnd)
- **概要**: アトラシアン（Atlassian）が開発していた有名な `react-beautiful-dnd` を、コミュニティがフォークして現在もメンテナンスを続けているライブラリ。**複雑なUIを持つシミュレータ等で利用実績が豊富な技術**です。
- **特徴**:
  - `DragDropContext`, `Droppable`, `Draggable` の3つの主要コンポーネントで構成されており、概念がわかりやすい。
  - アニメーションがデフォルトで非常に滑らか。
  - マウスだけでなく、キーボード操作やスクリーンリーダーにも対応（アクセシビリティが高い）。
- **用途**: リスト間のアイテム移動、キャラクターの順序入れ替え、編成画面の実装など。

### 候補2: [dnd-kit](https://dndkit.com/)
- **概要**: よりモダンで軽量な、拡張性の高いドラッグ＆ドロップツールキット。
- **特徴**:
  - モジュール化されており、必要な機能・センサー（マウス、タッチ、キーボード等）だけをインポートできるため、バンドルサイズを抑えられる。
  - `react-beautiful-dnd` よりもカスタマイズ性が高く、物理ベースのアニメーションなども柔軟に実装可能。
- **用途**: `hello-pangea/dnd` で要件を満たせないような複雑なドラッグ＆ドロップコンポーネントを作る場合、または新規プロジェクトで最新のライブラリを採用したい場合。

### 実装の基本概念 (hello-pangea/dnd の場合)
1. **`DragDropContext`**: 操作対象となる全体エリアをラップし、`onDragEnd` イベント（ドロップ完了時のデータ更新処理）を受け取る。
2. **`Droppable`**: アイテムをソート/ドロップできるリスト領域（例：前衛エリア、後衛エリア）。`droppableId` を割り当てる。
3. **`Draggable`**: ドラッグして移動させる個別の要素（例：キャラクターアイコン）。`draggableId` と `index` を割り当てる。

## 3. 今後の開発への活かし方

- **編成シミュレータ等のリファクタリング・新規作成**:
  現在のUIが扱いづらい場合や、より直感的にキャラクターを配置したい場合、`React` + `@hello-pangea/dnd` (または `dnd-kit`) を導入することで、ユーザー体験（UX）を劇的に向上させることが可能です。
- **状態管理**:
  ドラッグ＆ドロップで要素の並びが変わった際に、その順序データ（配列）をReactのStateや、Redux・Zustandといった状態管理ライブラリと同期させることで、内部のシミュレーションロジックとUIを完全に分離・連携させることができます。
