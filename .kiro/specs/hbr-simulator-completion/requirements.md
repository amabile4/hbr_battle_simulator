# HBRシミュレータ完成化プロジェクト 要求仕様書

## 概要

既存のヘブンバーンズレッド戦闘シミュレータの未実装機能を完成させ、完全な戦術分析ツールとして仕上げるプロジェクト。

## 要求事項

### 要求1: タブ区切りデータ出力機能の実装

**ユーザーストーリー:** 戦術分析者として、既存のスプレッドシートテンプレートに戦闘データを貼り付けて分析したい

#### 受入基準
1. WHEN ユーザーが「データ出力」ボタンをクリック THEN システムはタブ区切り形式でデータを出力する
2. WHEN データ出力を実行 THEN 2行ヘッダー構造（キャラクター名行、列ヘッダー行「始・行動・終」）で出力される
3. WHEN 特殊ターンが記録されている THEN 「OD1」「OD2」「OD3」「追加」等の表記でT列に記録される
4. WHEN SP変動がある THEN 各キャラクターの「始」「行動」「終」列に正確に記録される
5. WHEN 出力データをコピー THEN 既存スプレッドシートに直接貼り付け可能である

### 要求2: オーバードライブシステムの実装

**ユーザーストーリー:** 戦術プランナーとして、オーバードライブを含む高度な戦術をシミュレートしたい

#### 受入基準
1. WHEN ユーザーがOD1を発動 THEN 発動者のSPが+5回復し、追加ターン+1が付与される
2. WHEN ユーザーがOD2を発動 THEN 発動者のSPが+12回復し、追加ターン+2が付与される
3. WHEN ユーザーがOD3を発動 THEN 発動者のSPが+20回復し、追加ターン+3が付与される
4. WHEN オーバードライブでSP回復 THEN 通常上限20を超えて回復可能である
5. WHEN オーバードライブゲージが100%以上 THEN OD発動ボタンが有効になる

### 要求3: 追加ターンシステムの実装

**ユーザーストーリー:** 戦術研究者として、追加ターンを含む複雑な戦術パターンを検証したい

#### 受入基準
1. WHEN 特定スキルで追加ターンが発生 THEN 該当キャラクターに追加ターン状態が付与される
2. WHEN 追加ターン状態のキャラクターがいる THEN 配置交代は追加ターン状態のキャラクター間でのみ可能である
3. WHEN 初回追加ターン THEN 該当キャラクターのSPが+5回復する
4. WHEN 追加ターンを実行 THEN ターン表記が「追加1」「追加2」等で記録される

### 要求4: 設定画面の実装

**ユーザーストーリー:** シミュレータ利用者として、デフォルトパーティーや基本設定をカスタマイズしたい

#### 受入基準
1. WHEN ユーザーが設定画面を開く THEN デフォルトパーティーの6キャラクターを変更できる
2. WHEN ユーザーが設定を変更 THEN 次回起動時に変更された設定が適用される
3. WHEN ユーザーが初期SP設定を変更 THEN 新規パーティー作成時に反映される
4. WHEN ユーザーが設定をリセット THEN 工場出荷時の設定に戻る

### 要求5: バフ・デバフ詳細管理の実装

**ユーザーストーリー:** 戦術分析者として、バフ・デバフの詳細な状態変化を追跡したい

#### 受入基準
1. WHEN バフ・デバフスキルを使用 THEN 対象の状態が正確に記録される
2. WHEN エンハンス（攻撃力アップ）を重複使用 THEN 2回まで重ね掛けが記録される
3. WHEN バフ・デバフに継続ターン数がある THEN ターン経過で自動的に減少する
4. WHEN 状態異常解除スキルを使用 THEN 該当する状態が除去される

### 要求6: 配置依存SP回復の実装

**ユーザーストーリー:** 高度な戦術プランナーとして、配置による複雑なSP回復効果を活用したい

#### 受入基準
1. WHEN キャラクターが前衛配置でターン開始 THEN 配置依存のSP回復効果が適用される
2. WHEN キャラクターが後衛配置でターン開始 THEN 後衛専用のSP回復効果が適用される
3. WHEN パッシブスキルで他者にSP回復効果がある THEN 条件に応じて他キャラクターのSPが回復する
4. WHEN 複数のSP回復効果が重複 THEN すべての効果が正しく累積される

### 要求7: 行動順序実行システムの実装

**ユーザーストーリー:** 戦術研究者として、実際の戦闘と同じ行動順序でスキル効果を確認したい

#### 受入基準
1. WHEN ターンを実行 THEN 非ダメージスキルがポジション順（1→2→3）で先に実行される
2. WHEN 非ダメージスキル実行完了 THEN ダメージスキルがポジション順（1→2→3）で実行される
3. WHEN スキル効果でSP回復がある THEN 実行順序に従って段階的にSPが変動する
4. WHEN バフ・デバフ効果がある THEN 行動順序に応じて状態が変化する
