# ヘルプドキュメント

このディレクトリは、ヘルプページから取得した情報をもとに生成された仕様書作成用のMarkdownファイルを格納しています。
シミュレータの実装に関わる情報は、「キャラクター」および「バトル」カテゴリに追加調査と実装メモを記載します。

## カテゴリごとの追加調査ステータス

- **キャラクター**: ◯ (追加調査を行う)
- **バトル**: ◯ (追加調査を行う)
- **Google_Play_Games_on_PC**: × (追加調査を行わない)
- **Steam版**: × (追加調査を行わない)
- **アプリ全般**: × (追加調査を行わない)
- **クォーツ**: × (追加調査を行わない)
- **ストーリー**: × (追加調査を行わない)
- **プレミアムパス・ライトパス**: × (追加調査を行わない)
- **メニュー**: × (追加調査を行わない)
- **動作不良**: × (追加調査を行わない)

## テンプレート自動追記スクリプトについて

一部の機能説明Markdown（例: `バトル` や `キャラクター` カテゴリ）に対して、シミュレータの実装情報（内部名や、対応スキル・パッシブ一覧）を JSONデータ から自動抽出・追記するためのスクリプトを用意しています。

今度の機能追加やゲーム内ヘルプの仕様説明に変更があった場合は、以下のスクリプトを再実行することで自動的にJSONの記述内容をマージし直すことが可能です。

### スクリプトの場所

- `help/scripts/populate_templates.py`

### 使い方

ルートディレクトリから以下のコマンドを実行します：

```bash
python3 help/scripts/populate_templates.py
```

### 注意事項

- スクリプトは `data/json/skills.json` および `data/json/passives.json` の `desc` フィールドに対して、Markdownのファイル名（またはエイリアス）での**部分一致検索**を行います。
- `EXスキル連続発動.md` や `BREAK.md` のように、手動でエイリアス検索などの修正を入れた特別ファイルについては、スクリプト内で `if keyword in [...]` の形でスキップ処理が記述されています。
- 自動追記がうまくいかないものは「ヒットなし」として書き出されます。その場合はファイルを開いて直接手動で実装メモを書き込んでください。
