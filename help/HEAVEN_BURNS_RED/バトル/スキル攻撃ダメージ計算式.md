# スキル攻撃ダメージ計算式 仕様書

## 1. 計算式表現（数式定義）

最終ダメージ = ベース値 * 基礎_属性バフ枠 * 敵のデバフ枠 * 連撃バフ枠 * トークンバフ枠 * クリティカル補正枠 * 武器相性枠 * 属性相性枠 * 破壊率_特効_ランダム枠

各枠の計算式は以下の通り定義する。

[ベース値]
ベース値 = スキル攻撃力

[基礎_属性バフ枠]
基礎_属性バフ枠 = 1.0 + スキル攻撃力バフ + スキル属性攻撃力バフ + チャージ + アクセサリ + パッシブスキル他 + 心眼バフ + 属性フィールド

[敵のデバフ枠]
敵のデバフ枠 = 1.0 + 防御ダウン + 永続防御ダウン + 属性防御ダウン + 永続属性防御ダウン + DP防御ダウン + パッシブスキル他 + 脆弱 + 永続脆弱

[連撃バフ枠]
連撃バフ枠 = 1.0 + 連撃バフ + パッシブスキル他

[トークンバフ枠]
トークンバフ枠 = 1.0 + トークンバフ

[クリティカル補正枠]
・クリティカル発生時
クリティカル補正枠 = 1.5 + クリ威力 + 属性クリ威力 + パッシブスキル他
・非クリティカル発生時
クリティカル補正枠 = 1.0

[武器相性枠]
武器相性枠 = 敵の武器属性相性倍率（斬・突・打）

[属性相性枠]
属性相性枠 = 1.0 + 属性相性値 + 属性耐性ダウン

[破壊率_特効_ランダム枠]
破壊率_特効_ランダム枠 = 現在の破壊率 * (1.0 + 特効倍率) * 乱数

## 2. 各項目の補足説明

### ■ ベース値
* **スキル攻撃力**: キャラクターのステータスおよびスキル固有の威力から算出される基礎値。※クリティカル時は敵能力値を「-50」として計算する仕様が別途存在する。

### ■ 基礎・属性バフ枠
* **スキル攻撃力バフ / スキル属性攻撃力バフ**: スキル効果によって上昇する各種攻撃バフ値。
* **チャージ**: チャージスキルの効果値。
* **アクセサリ**: 装備している各種リングやピアスの効果値。
* **パッシブスキル他**: アビリティやスタイルパッシブなどの効果値。
* **心眼バフ**: 心眼によって付与されるバフ値。
* **属性フィールド**: 属性強化フィールド（弱:0.35 / 通常:0.65 / 大:0.80 など）の効果値。

### ■ 敵のデバフ枠
* **防御ダウン / 永続防御ダウン**: 敵に付与されている防御デバフ値。
* **属性防御ダウン / 永続属性防御ダウン**: 敵に付与されている属性防御デバフ値。
* **DP防御ダウン**: 敵のDPに対する防御デバフ値。
* **パッシブスキル他**: 敵の弱体化に関わるパッシブ効果値。
* **脆弱 / 永続脆弱**: 敵に付与されている脆弱の効果値。

### ■ 連撃バフ枠
* **連撃バフ**: 連撃数（連撃数上昇バフ）による一撃あたりの重み付け補正値。

### ■ トークンバフ枠
* **トークンバフ**: 特定のスタイルが持つ「トークン」の蓄積数に応じた倍率補正値。

### ■ クリティカル補正枠
* **クリ威力 / 属性クリ威力**: クリティカル時の威力を増加させるバフ・スキル効果値。

### ■ 武器相性枠
* **敵の武器属性相性倍率**: 敵の弱点・耐性（等倍:1.0 / 弱点:1.5 / 耐性:0.5 など）。

### ■ 属性相性枠
* **属性相性値**: 元素属性（火・水・雷・氷・光・闇）の弱点・耐性値。
* **属性耐性ダウン**: エレメンタルダウン等、デバフとは別枠で加算される属性耐性ダウン値。

### ■ 破壊率＆特効＆ランダム枠
* **現在の破壊率**: 敵の現時点の破壊率（100%〜999%を内部的に 1.0〜9.99 として扱う）。
* **特効倍率**: スキルが持つ「HP+50%」などの特定状況下の特効効果値（DP攻撃時はDP特効、HP攻撃時はHP特効を適用）。
* **乱数**: 0.9 〜 1.1 の間でランダムに変動する最終ブレ幅。

## 3. データ定義仕様

ゲーム内およびシミュレーター内部のJSONデータ（`skills.json`, `passives.json` など）で定義されているバフ・デバフの `skill_type` やパラメータと、ダメージ計算式の各枠との対応関係は以下の通りです。

### 1. 基礎・属性バフ枠
計算式：`1.0 + スキル攻撃力バフ + スキル属性攻撃力バフ + チャージ + アクセサリ + パッシブスキル他 + 心眼バフ + 属性フィールド`

| 枠内の項目 | 内部データ定義（`parts[].skill_type` / 条件） | 効果値 `power`（基本値）の範囲 | 日本語表示名 (`STATUS_LABELS`) | 補足説明・適用仕様 |
| :--- | :--- | :--- | :--- | :--- |
| **スキル攻撃力バフ** | `AttackUp`<br>（`elements` が空配列かつ、対象がスキル） | `0.05 〜 3.0`<br>(`+5% 〜 +300%`) | 攻撃力アップ | 通常のスキル攻撃力を上昇させるバフ。通常発動（2回合算）か、単独発動（1つのみ採用）に分かれます（詳細は「4. バフの適用形式」を参照）。 |
| **スキル属性攻撃力バフ** | `AttackUp`<br>（`elements` に属性元素が含まれる場合） | `0.05 〜 3.0`<br>(`+5% 〜 +300%`) | 属性攻撃力アップ<br>（例: 火攻撃力アップ 等） | 特定の属性（元素属性または武器属性）を伴う攻撃の時のみ適用される攻撃力バフ。通常発動・単独発動が存在します。 |
| **チャージ** | `BuffCharge` | `0.2 〜 0.3`<br>(`+20% 〜 +30%`) | チャージ | スキル使用時に消費される、次のスキル威力を上昇させるバフ。ダメージ計算の基礎・属性バフ枠に加算されます。 |
| **パッシブスキル他** | `AttackUp` / `AttackUpPerToken`<br>（パッシブスキル・アビリティ由来） | `0.05 〜 0.2`<br>(`+5% 〜 +20%`) | 攻撃力アップ<br>（アビリティ/スタイルパッシブ） | キャラクターの限界突破アビリティやスタイルパッシブ（例：前衛にいるとき攻撃力+10%など）によるバフ値。 |
| **心眼バフ** | `MindEye` | `0.42 〜 1.2`<br>(`+42% 〜 +120%`) | 心眼 | 敵の弱点属性を突いた時のダメージを上昇させるバフ。単独発動タイプが多くを占めます。 |
| **属性フィールド** | `Zone` / `ZoneUpEternal` | `0.35 〜 0.95`<br>(`+35% 〜 +95%`) | フィールド（例: 火属性フィールド 等） | バトルフィールド全体に展開される属性強化フィールド。パッシブスキルによる強化効果も合算されます（詳細は下記詳細記述を参照）。 |

#### ■ 属性フィールドの威力計算詳細
属性フィールドは、バトル全体の環境を特定の属性有利に変化させ、一致する属性攻撃の威力を割合で上昇させます。

1. **基本フィールド効果値 (`Zone`)**:
   スキルによって展開される初期フィールドの倍率は以下の通りです。
   * **弱フィールド**: `0.35` (`+35%`、ゲームデータ上は `1.35`倍)
   * **通常フィールド**: `0.65` (`+65%`、ゲームデータ上は `1.65`倍)
   * **大フィールド**: `0.80` (`+80%`、ゲームデータ上は `1.80`倍)

2. **パッシブによるフィールド強化 (`ZoneUpEternal`)**:
   一部のスタイルが所持するパッシブスキル（例：「フィールド展開時、対象属性のスキル攻撃力+15%」および「フィールドの永続化」）により、フィールドの効果値がさらに強化されます。
   * 強化ボーナス値：一律で **`0.15`** (`+15%`)

3. **計算具体例**:
   * **通常の大フィールドのみ**: フィールド効果 ＝ `0.80` (`+80%`)
   * **大フィールド ＋ フィールド強化パッシブ**: フィールド効果 ＝ `0.80 + 0.15 = 0.95` (`+95%`)

### 2. 敵のデバフ枠
計算式：`1.0 + 防御ダウン + 永続防御ダウン + 属性防御ダウン + 永続属性防御ダウン + DP防御ダウン + パッシブスキル他 + 脆弱 + 永続脆弱`

| 枠内の項目 | 内部データ定義（`parts[].skill_type` / 条件） | 効果値 `power`（基本値）の範囲 | 日本語表示名 (`STATUS_LABELS`) | 補足説明・適用仕様 |
| :--- | :--- | :--- | :--- | :--- |
| **防御ダウン / 永続防御ダウン** | `DefenseDown`<br>（`elements` が空配列かつ、対象が敵全体/単体） | `0.1 〜 1.0`<br>(`+10% 〜 +100%`) | 防御力ダウン | 敵の防御力を下げるデバフ。ターン制限のあるものと、戦闘中永続するもの（永続デバフ）が存在します。 |
| **属性防御ダウン / 永続属性防御ダウン** | `DefenseDown`<br>（`elements` に属性元素 `Fire`, `Ice` などが含まれる場合） | `0.1 〜 1.0`<br>(`+10% 〜 +100%`) | 属性防御力ダウン<br>（例: 火防御力ダウン 等） | 敵の特定属性に対する防御力を下げるデバフ。攻撃属性と一致する場合にのみ適用され、**「敵のデバフ枠」に直接加算**されます。 |
| **脆弱 / 永続脆弱** | `Fragile` | `0.3 〜 0.42`<br>(`+30% 〜 +42%`) | 脆弱 | 敵が弱点属性で受けるダメージを増幅するデバフ。弱点属性攻撃時のみ適用されます。 |

---

### 3. 連撃バフ枠
計算式：`1.0 + 連撃バフ + パッシブスキル他`

| 枠内の項目 | 内部データ定義（`parts[].skill_type`） | 効果値 `power`（連撃バフ威力）の範囲 | 日本語表示名 (`STATUS_LABELS`) | 補足説明・適用仕様 |
| :--- | :--- | :--- | :--- | :--- |
| **連撃バフ** | `Funnel` | `0.18 〜 1.5`<br>(`+18% 〜 +150%`) | 連撃数アップ | 連撃数上昇バフ。ヒット数増加と威力増加の両面を持ち、ダメージ計算に寄与する威力（効果値）は「追加ヒット数 * 規模別ボーナス」として算出されます。 |

#### ■ 連撃バフの威力・ヒット数計算詳細
連撃バフは、攻撃の**追加ヒット数**を増やす効果と、それに伴う**与ダメージの威力（ダメージ倍率）を上昇させる効果**を併せ持ちます。

1. **威力上昇（ダメージボーナス）の計算式**:
   `連撃バフ = 連撃数（追加ヒット数） * 1ヒットあたりの威力ボーナス`
   ※ 1ヒットあたりの威力ボーナスはデータ上の `part.value[0]` または `metadata.damageBonus` に定義されている割合です。

2. **連撃規模（小・中・大・特大）ごとのボーナス割合**:
   * **小**: `0.06` (`6%`)
   * **中**: `0.12` (`12%`)
   * **大**: `0.25` (`25%`)
   * **特大**: `0.50` (`50%`)

3. **計算具体例**:
   * **連撃（小）5回**: 追加ヒット数 +5回、連撃バフ威力 = `5 * 0.06 = 0.30` (`+30%`)
   * **連撃（大）3回**: 追加ヒット数 +3回、連撃バフ威力 = `3 * 0.25 = 0.75` (`+75%`)
   * **連撃（特大）3回**: 追加ヒット数 +3回、連撃バフ威力 = `3 * 0.50 = 1.50` (`+150%`)

4. **競合・合算ルール**:
   通常発動（最大2枠の合算）と単独発動（最大効果の1枠）でより効果が高いバフグループが優先採用されます（詳細は「4. バフの適用形式」を参照）。
   * **通常発動同士の合算例**: 通常発動の「連撃(小)5回」と「連撃(中)3回」が同時にかかっている場合、合算されて `連撃数 +8回`、`連撃バフ威力 +66% (30% + 36%)` になります。
   * **通常発動と単独発動の競合例**: 通常合算の「連撃数+8/威力+66%」と、単独発動の「連撃(特大)3回/威力+150%」が同時にかかっている場合、威力効果が高い単独発動が優先採用され、結果は `連撃数 +3回`、`連撃バフ威力 +150%` となります。

### 4. トークンバフ枠
計算式：`1.0 + トークンバフ`

| 枠内の項目 | 内部データ定義（`parts[].skill_type`） | 効果値 `power`（トークンバフ威力）の範囲 | 日本語表示名 (`STATUS_LABELS`) | 補足説明・適用仕様 |
| :--- | :--- | :--- | :--- | :--- |
| **トークンバフ** | `TokenSet` で獲得したトークンを利用した倍率補正（`TokenAttack`） | `0.0 〜 1.6`<br>(`+0% 〜 +160%`) | トークン上昇 / トークン | トークン関連のスキルでのみ計算に使用されます（詳細は下記詳細記述を参照）。それ以外のスキルでは `0.0`（等倍）として扱われます。 |

#### ■ トークンバフの威力計算詳細
トークンバフは、特定のスタイルが所持する「トークン」の蓄積数に応じて、対応する特効スキル（内部的には `TokenAttack` に分類されるスキル）の威力を上昇させます。

1. **トークンバフ威力（ダメージボーナス）の計算式**:
   `トークンバフ = トークン所持数 * トークン1個あたりの威力上昇割合`
   * トークン所持数は最大10個です（`0` 〜 `10`）。
   * トークン1個あたりの威力上昇割合は、内部データ定義上は一律で `0.16` (`16%`) に設定されています。

2. **計算具体例**:
   * **トークン0個の場合**: `0 * 0.16 = 0.0` (`+0%`、トークンバフ枠は `1.0`＝等倍)
   * **トークン5個の場合**: `5 * 0.16 = 0.80` (`+80%`、トークンバフ枠は `1.8`＝1.8倍)
   * **トークン10個（最大）の場合**: `10 * 0.16 = 1.60` (`+160%`、トークンバフ枠は `2.6`＝2.6倍)

| :--- | :--- | :--- | :--- | :--- |
| **属性相性値** | 敵データの基本耐性倍率 | `-0.5 〜 +0.5`<br>(等倍: 0.0 / 弱点: +0.5 / 耐性: -0.5など) | 各属性の弱点・耐性 | 敵が元々持っている属性相性（耐性: 0.5倍 / 等倍: 1.0倍 / 弱点: 1.5倍）を基本値（1.0）からの増減値として扱います。 |
| **属性耐性ダウン** | `ResistDown` / `ResistDownOverwrite` | `0.2 〜 0.3`<br>(`+20% 〜 +30%`) | 属性耐性ダウン<br>(エレメンタルダウン) | 敵の属性耐性を下げるデバフ。属性防御ダウンとは異なり、**「属性相性枠」内で敵の基本属性相性に加算**されます。 |

## 4. バフの適用形式（通常発動と単独発動）の分類

バトル中のダメージ計算において、バフ効果（スキル攻撃力バフ、属性攻撃力バフ、連撃バフ、クリティカルダメージアップなど）は、スキルの適用形式（通常発動と単独発動）に応じて解決ロジックが異なります。シミュレーター内部およびゲーム仕様との突き合わせは以下の通りです。

### ■ 通常発動（通常バフ / 回数制限バフ）
* **スキル説明での表現**: 「スキル攻撃力を上げる」などの一般的なバフ表記（特に回数が指定されるか、ターン数のみの永続でないもの）。
* **データ定義 (`skills.json`)**: `parts[].effect.limitType` が `Default` 等で、`exitCond` が `Count`（回数制限）に設定されているもの。
* **ゲーム内＆シミュレーター実装挙動**:
  * 同種バフ（例：スキル攻撃力アップ）が複数付与されている場合、効果値（`power`）が高いものから**最大2つまで合算**されます（2重の重ねがけ合算が可能）。
  * 適用時には回数（`remaining`）が消費されます。

### ■ 単独発動（専用限定バフ）
* **スキル説明での表現**: スキル説明文の中に **`[単独発動]`** の表記があるもの。
  * 例：「3ターンの間 自身のスキル攻撃力を著しく上げ[単独発動]」など。
* **データ定義 (`skills.json`)**: `parts[].effect.limitType` が **`Only`**（専用スキル限定バフ）に設定されているもの。
* **ゲーム内＆シミュレーター実装挙動**:
  * 複数付与されていても、**最も効果が高い1つの効果値のみ**が採用され、重複して合算されることはありません。
  * 通常、ターン経過（`exitCond` が `PlayerTurnEnd` など）で管理されます。

### ■ シミュレーターでの解決 (Resolution) と評価ルール
シミュレーターのバフ解決関数（[turn-controller.js](file:///Users/ram4/git/hbr_battle_simulator/src/turn/turn-controller.js#L7114-L7150)）では、現在付与されているバフから通常発動（Count）と単独発動（Only）を抽出し、以下のルールで競合解決を行います。

1. **通常発動枠（合算）と単独発動枠（単発）の比較**:
   * 通常発動（Countバフ）の上位2つの合算値と、単独発動（Onlyバフ）の最大値のうち、**より効果値が高い方のバフグループが優先採用**されます。
   * 計算式：`adopted = 通常発動の合算値(最大2枚) >= 単独発動の最大値(1枚) ? 通常発動を採用 : 単独発動を採用`
2. **永続バフとの合算**:
   * 上記で採用されたバフグループの効果値は、限界突破アビリティなどの「永続バフ」および「パッシブスキル他」とさらに合算されます。
3. **スキルとパッシブの共存**:
   * 「アクティブスキルによる単独発動バフ」と「パッシブスキルによるバフ」は別枠であり、それぞれ独立して効果を発揮します。

## 5. 威力増加量の計算手順と表示仕様（UI向け）

スキルを実行した際に適用される各種バフ・デバフによる「威力増加量」を算出し、その根拠と計算式をUI上に視覚的に出力するための定義です。

### 1. 増加率の算出計算式
ベース値（バフ等なしの等倍状態 ＝ `1.0`）に対する、バフ・デバフの効果を乗算した合計増加率（％）は以下の式で計算します。
※ ユーザー指示に基づき、乱数は常に `1.0` 固定（計算上除外）とします。

```
合計増加率 (%) = ( 基礎_属性バフ枠 * 敵のデバフ枠 * 連撃バフ枠 * トークンバフ枠 * クリティカル補正枠 * 武器相性枠 * 属性相性枠 * 破壊率_特効枠 - 1.0 ) * 100
```

### 2. UI表示する構成要素
UI上に出力する「威力詳細パネル」では、以下の要素を出力します。

#### ① 合計の威力（最終倍率と増加量）
* 最終合計倍率（例: `3.56倍`）
* ベースに対する純増加量（例: `+256%`）

#### ② 計算式と各枠の合計パーセント
数式を展開して表示します。
* 表示例: `最終威力 = 1.0 * 2.10 (バフ枠) * 1.30 (デバフ枠) * 1.30 (連撃枠) * 1.00 (トークン枠) * 1.50 (クリ補正枠) * 1.50 (弱点相性枠) = 7.17倍 (+617%)`

#### ③ 根拠バフ・デバフとアイコンの対応（内訳表示）
各枠の中に含まれる個別のバフをリストアップします。
* **表示名・アイコンアセットの解決**:
  `STATUS_LABELS` に定義されている日本語表示名と、右クリックウィンドウで解決されているアイコン画像定義（`resolveSkillTypeIconUrl` で取得できるアセットイメージ）をそのまま流用して表示します。
* **パッシブアビリティの表現**:
  現時点では「パッシブスキル名（例: [前衛]火属性のスキル攻撃力+10%）」と「効果値（例: `+10%`）」のテキスト表示として解決します。
* **アクセサリ**:
  現時点では未実装とし、威力表示の計算および表示対象から除外（または `0%` として無視）します。
