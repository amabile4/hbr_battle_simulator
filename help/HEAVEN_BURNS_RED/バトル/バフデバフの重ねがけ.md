- **カテゴリ**: HEAVEN BURNS RED > バトル

## ヘルプ記載内容
バフ/デバフの重ねがけ
バトル中に付与されるバフ/デバフには、重ねがけをできるものがあります。※重ねがけをした際に効果が発揮される個数は、種類によって異なります。また、発揮される個数より多くの効果が重ねがけされている場合、効果が高いものから優先して効果が発揮されます。※同じ名称のバフ/デバフ効果でも、有効期間が異なる場合、効果がそれぞれ発揮されます。

## シミュレーター表示ルール

### 概要

バフ/デバフの重ねがけ表示は以下の3か所で使用される。

| 表示箇所 | 対象 | 表示形式 | 備考 |

| buff-icon-list | 味方バフ | アイコンのみ（最大10個） | ターン行内のアイコン列 |

### 1. 有効/無効の判定

#### 1.1 存在判定（アクティブ＝期限切れでない）

効果が「存在している」かどうかの判定。期限切れ効果は表示対象外。

| 条件 | 判定 |
|---|---|
| exitCond = 'Eternal' | アクティブ |
| remaining > 0 | アクティブ |
| 敵永続型（Break, SuperBreak, Dead 等） | アクティブ |
| 上記いずれにも該当しない | 非アクティブ → **表示しない** |

> 実装: `buff-display.js#isActiveEffect()`, `enemy-status-display.js#isActiveEnemyStatus()`

#### 1.2 採用判定（重ねがけ時の有効/無効）

アクティブな効果であっても、同種の効果が重ねがけされている場合、ゲーム仕様により「**採用（有効）**」と「**非採用（無効）**」に分かれる。

- **採用（有効）**: 現在バトル計算に反映されている効果
- **非採用（無効）**: 存在しているが重ねがけ上限超過により計算に反映されない効果（上位効果が切れれば自動的に有効になる）

#### 1.2.1 採用グループキー

同一グループ内で採用競合が行われる。グループキーは以下の組み合わせ:

```
グループキー = 状態変化種別(statusType) × 属性(elements) × 期間グループ
```

| 要素 | 説明 |
|---|---|
| statusType | 効果の種別（DefenseDown, Fragile 等） |
| elements | 属性（Fire, Ice, Thunder, Light, Dark）。属性なしと属性付きは別グループ |
| 期間グループ | **無限グループ** / **有限グループ** の2区分（下記参照） |

**期間グループの分類**:

| 期間グループ | 該当する exitCond | 説明 |
|---|---|---|
| **無限グループ** | Eternal | 永続効果。期限なし。∞ 表記 |
| **有限グループ** | Turn, Count | ターン経過で消滅するもの＋スキル消費で残数減少するもの。両者は同一枠で競合する |

> **ゲーム仕様との対応**: 「同じ名称のバフ/デバフ効果でも、有効期間が異なる場合、効果がそれぞれ発揮されます」→ 無限グループと有限グループでそれぞれ独立した採用枠を持つ。

#### 1.2.2 グループ内の採用上限

各グループ（無限/有限）内での採用上限はエンジンの `limitType` で決まる:

| limitType | 採用上限 | 選択基準 |
|---|---|---|
| Default | **全て採用** | — |
| Only | **1個** | 効果値(power)最大のもの |
| Count | **2個** | 効果値(power)上位2個 |

- 無限グループの ∞ 効果であっても、limitType に基づき採用上限が適用される（例: limitType=Only の ∞ 効果は同一グループ内で最大1個のみ採用）
- Only vs Count の競合: 同一グループ内に Only と Count の両方が存在する場合、効果値の合計が大きい方を採用
- 同一 power の場合は remaining が大きいもの、さらに同値なら effectId 昇順で選択

> **ゲーム仕様との対応**: 「効果が発揮される個数は種類によって異なります」→ limitType がこれを制御。「効果が高いものから優先して効果が発揮されます」→ power 降順で選択。

> 実装: `buff-display.js#selectAdoptedEffectsByCompetition()`, `character-style.js#resolveEffectiveStatusEffects()`

#### 1.2.3 既知の不整合（要対応）

| 箇所 | 現状 | 仕様 |
|---|---|---|
| buff-display.js | statusType のみでグループ化 | statusType × elements でグループ化すべき |
| char-detail-popup.js | 採用/非採用の区別なし（全アクティブ効果を等しく表示） | 採用/非採用を視覚的に区別 |
| enemy-status-display.js | 採用/非採用の区別なし | 採用/非採用を視覚的に区別 |

> エンジン層 (`character-style.js#getStatusEffectOnlyGroupKey()`) は `elements` を含めたグループキーで競合判定しており、仕様と整合している。表示層を合わせる必要がある。

### 2. ソート順序

#### 2.1 大分類: バフ/デバフ順

表示箇所によりバフ/デバフの表示優先が異なる:

| 表示箇所 | 順序 | 理由 |
|---|---|---|
| char-popup-panel | **バフ → デバフ** | 味方キャラクターでは有利効果を優先表示 |
| enemy-detail-popup | **デバフ → バフ** | 敵に対してはプレイヤーが付与したデバフを優先表示 |
| buff-icon-list | バフのみ | デバフは表示対象外（既存どおり） |

#### 2.2 中分類: 属性バリアント分類

大分類（バフ/デバフ）内で、属性バリアントの有無に基づきさらに分類する。

| カテゴリ | 説明 | 該当する種別の例 |
|---|---|---|
| **(1)** | 属性あり・属性なし**両方が存在しうる**種別 | AttackUp/Down, DefenseUp/Down, CriticalRateUp/Down, CriticalDamageUp/Down, ResistDown/Up |
| **(1)a** | ↑のうち属性付き効果 | 闇防御力ダウン, 闇耐性ダウン, 闇攻撃力アップ |
| **(1)b** | ↑のうち属性なし効果 | 防御力ダウン, 耐性ダウン, 攻撃力アップ |
| **(2)** | 属性バリアントが**存在しない**種別 | Fragile, HealDown, BuffCharge, MindEye, Funnel 等 |

表示順: **(1)a → (1)b → (2)**

> 「属性バリアントが存在しうる種別」の判定は `ELEMENT_PREFIXED_STATUS_TYPES`（element-status-constants.js）から基底 statusType を抽出して決定する。ResistDownOverwrite のように属性付きのみ存在する種別も (1)a に含める。

#### 2.3 小分類: 種別内ソート

同一カテゴリ内では skill_types.json の ID 昇順（現行 STATUS_TYPE_ID_MAP / ENEMY_STATUS_TYPE_ID_MAP / STATUS_TAB_SKILL_TYPE_ID_MAP）で種別を並べる。

#### 2.4 種別内の個別効果ソート

同一種別（statusType × elements が一致）の効果群は以下の優先度でソートする:

| 優先度 | ソートキー | 方向 | 説明 |
|---|---|---|---|
| 1 | 期間グループ | 無限 → 有限 | 永続効果が最優先 |
| 2 | 採用状態 | 採用 → 非採用 | 有効な効果を上位に表示 |
| 3 | 期間区分（非採用内） | ターン制 → 回数制 | 非採用効果の中での副次ソート（※後述） |
| 4 | 効果値(power) | 降順 | 高い方が先 |
| 5 | 残りターン/回数(remaining) | 降順 | 多い方が先 |
| 6 | effectId | 昇順 | タイブレーカー |

**優先度3（ターン制→回数制）の補足**:
- 有限グループ内の**非採用効果**に対してのみ適用する副次ソート
- ターン制（exitCond=Turn）の方が実装時期が後期で採用されやすい傾向があるため、非採用効果の中でもターン制を先に表示する
- 採用効果内ではこの区分は適用せず、直接 power 順でソートする

#### 2.5 ソート全体の適用順まとめ

```
1. バフ/デバフ分類        (char-popup: バフ先 / enemy-popup: デバフ先)
  2. 属性バリアント分類    (1)a → (1)b → (2)
    3. 種別ID順            skill_types.json ID 昇順
      4. 期間グループ       無限 → 有限
        5. 採用状態         採用 → 非採用
          6. 期間区分        [非採用内] ターン制 → 回数制
            7. 効果値        降順
              8. remaining   降順
                9. effectId  昇順
```

### 4. 想定例

敵に対する状態変化の表示例（enemy-detail-popup テーブル、デバフのみ抜粋）:

```
○  (1)a  DefenseDown       闇防御力ダウン +40%     [ヒットチャートからの一閃] 逢川めぐみ   ∞
○  (1)a  DefenseDown       闇防御力ダウン +45%     [くぎづけ♡ラブリービーム] 蒼井えりか   2T
○  (1)a  ResistDown        闇耐性ダウン +60%       [今宵、快楽ナイトメア] 東城つかさ       ∞
   (1)a  ResistDown        闇耐性ダウン +60%       [今宵、快楽ナイトメア] 東城つかさ       ∞
   (1)a  ResistDown        闇耐性ダウン +60%       [今宵、快楽ナイトメア] 東城つかさ       ∞
○  (1)a  ResistDownOW      闇属性耐性打ち消し       [今宵、快楽ナイトメア] 東城つかさ       ∞
○  (1)b  DefenseDown       防御力ダウン +30%       [ヒットチャートからの一閃] 逢川めぐみ   ∞
○  (1)b  DefenseDown       防御力ダウン +45%       [ハードブレード] 逢川めぐみ             1T
○  (1)b  DefenseDown       防御力ダウン +45%       [ハードブレード] 逢川めぐみ             1T
   (1)b  DefenseDown       防御力ダウン +30%       [くぎづけ♡ラブリービーム] 蒼井えりか   2T
○  (2)   Fragile           脆弱 +40%               [まだまだ行くで！] 逢川めぐみ           ∞
○  (2)   Fragile           脆弱 +40%               [まだまだ行くで！] 逢川めぐみ           ∞
○  (2)   Fragile           脆弱 +60%               [ネイキッドイレイザー] 逢川めぐみ       1T
○  (2)   Fragile           脆弱 +60%               [ネイキッドイレイザー] 逢川めぐみ       1T
○  (2)   HealDown          回復量ダウン +100%      [ヒットチャートからの一閃] 逢川めぐみ   ∞
```

- **○** = 採用（有効）: バトル計算に反映中
- **空白** = 非採用（無効）: 存在するが重ねがけ上限超過で計算対象外

**この例の解説**:

| グループ | 効果 | 期間グループ | 効果数 | 採用数 | 推定limitType | 説明 |
|---|---|---|---|---|---|---|
| (1)a 闇DefenseDown | +40% | 無限 | 1 | 1 | — | 単独のため全採用 |
| (1)a 闇DefenseDown | +45% | 有限(Turn) | 1 | 1 | — | 単独のため全採用 |
| (1)a 闇ResistDown | +60% ×3 | 無限 | 3 | **1** | Only | 上限1、同power同remainingのため先頭を採用 |
| (1)b DefenseDown | +30% | 無限 | 1 | 1 | — | 単独のため全採用 |
| (1)b DefenseDown | +45%×2, +30% | 有限(Turn) | 3 | **2** | Count | 上限2、power上位2件(+45%)を採用。非採用の+30%はターン制なので回数制より先に表示 |
| (2) Fragile | +40% ×2 | 無限 | 2 | 2 | Default | 全て採用 |
| (2) Fragile | +60% ×2 | 有限(Turn) | 2 | 2 | Default | 全て採用 |
| (2) HealDown | +100% | 無限 | 1 | 1 | — | 単独のため全採用 |

### 5. 確認事項

#### 解決済み

| ID | 項目 | 決定内容 |
|---|---|---|
| Q-BD1 | 期間グループ分類 | **2グループ制を採用**: 無限グループ(∞) と 有限グループ(ターン制+回数制)。有限グループ内ではターン制と回数制が同一枠で競合する。非採用効果の中ではターン制→回数制の順（ターン制の方が後期実装で採用されやすい傾向のため先に表示）。 |
