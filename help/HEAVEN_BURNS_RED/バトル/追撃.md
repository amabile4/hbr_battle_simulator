## ヘルプ記載内容
追撃
追撃は後衛にいるときに発動するアビリティです。前衛がスキル攻撃を行ったときに一定の確率で発動し、SPを消費しない無属性の攻撃を行います。発動確率は他のアビリティなどにより変動します。1回のスキル攻撃に対して発動する追撃は1回のみです。※前衛のスキル攻撃がMISSの場合でも発動します。※追撃キャラクターが行動不能などのデバフ効果を持つ場合でも発動します。※前衛のスキル攻撃によってターゲットしていた敵を撃破した場合、その攻撃への追撃は発動しません。※追撃はバトル中に付与されたスキル攻撃に対するバフ/デバフの効果を受けません。

## 追撃ヒット数の取得方法と OD 増加量計算の根拠

追撃の OD 増加量を計算するときは、追撃のヒット数を次のルールで取得して計算根拠にします。

### 前提
- `json/skills.json` にある exact name `追撃` の `hit_count` が、追撃ヒット数の一次情報です。
- `json/styles.json` の `weapon.type` は、追撃ヒット数のフォールバック判定に使えます。
- `追撃` の exact record は共通して `target_type=Single`, `consume_type=Sp`, `sp_cost=0`, `parts[0].skill_type=AttackNormal` です。
- 差分として重要なのは基本的に `hit_count` だけです。
- OD 増加量計算では、この `追撃ヒット数` を既存の「1Hit ごとの OD 増加量」計算に入力してください。

### 推奨ロジック
1. まず `skills.json` から、そのキャラ/スタイルに対応する exact name `追撃` レコードを探す。
2. 見つかったら、その `hit_count` を追撃ヒット数として採用する。
3. exact record が取れない場合だけ、`styles.json` の `weapon.type` から以下のフォールバック表で決める。
4. ただし `Gun` と `Scythe` には例外があるので、武器だけで固定しないこと。
5. さらに、大島四ツ葉の `温泉手形` により追撃が `ネコジェット・シャテキ` に変化する場合は、通常の `追撃` ではなく `ネコジェット・シャテキ` の `hit_count=5` を使う。

#### 💡 温泉手形による「ネコジェット・シャテキ」への変化条件と詳細仕様
大島四ツ葉のパッシブスキル『温泉手形』による追撃変化は、以下の厳密なルールに基づいて動作します。認識誤りが発生しやすいため注意してください。

* **消費SP条件（SP不足時のフォールバック）**
  * `ネコジェット・シャテキ` の発動には、**実効消費SP**（基本10SP、`ReduceSp` などの消費軽減パッシブが適用された後の値）以上の現在SPが追撃キャラクター（大島四ツ葉）に必要です。
  * SPが不足している場合、または同スキルを装備していない場合は、変化せずに**通常の「追撃」**が発動します。
* **「1回の行動中に1回のみ」の定義（ターン境界でのリセット）**
  * ゲーム内テキストに記載されている「1回の行動中に1回のみ」とは、シミュレーターにおける**「コミットされた1行（Committed Row / Turn Row）単位で1回のみ」**を指します。
  * **同一ターン行内（1回目と2回目の追撃）**:
    ビャッコ二連撃のような同一キャラクターの連続行動や、前衛の複数キャラクターのスキル使用によって、同じターン行内で複数回追撃がトリガーされる場合、**ネコジェットへの変化は最初の1回のみ**です。同じ行内の2回目以降の追撃は、SPが十分であっても通常の「追撃」になります。
  * **次のターン行への進行（EXターン/ODターンを含む）**:
    次のコミット行（追加ターンやオーバードライブターンを含む、新たなカード選択行）に進んだ場合は、再び最初の1回目に `ネコジェット・シャテキ` への変化が可能になります（使用済みフラグがリセットされます）。プレイヤーターン全体の終了（敵手番への移行）を待たずに、行単位でリセットされる仕様です。


### 簡易コード

```ts
type WeaponType =
  | "DoubleSword"
  | "LargeSword"
  | "Cannon"
  | "Shield"
  | "Claw"
  | "Sword"
  | "Gun"
  | "Scythe";

function getPursuitHitCount(ctx: {
  chara: string;
  style: string;
  weaponType: WeaponType;
  transformedPursuitSkillName?: string | null;
  skillsJson: Array<any>;
}): number {
  // 特殊変化: 大島四ツ葉の温泉手形
  if (ctx.transformedPursuitSkillName === "ネコジェット・シャテキ") {
    return 5;
  }

  // source of truth: exact name "追撃"
  const pursuit = ctx.skillsJson.find(
    (s) =>
      s.name === "追撃" &&
      s.chara === ctx.chara &&
      s.style === ctx.style
  );
  if (pursuit) {
    return pursuit.hit_count;
  }

  // fallback by weapon type
  switch (ctx.weaponType) {
    case "DoubleSword":
      return 2;
    case "LargeSword":
      return 2;
    case "Cannon":
      return 3;
    case "Shield":
      return 3;
    case "Claw":
      return 3;
    case "Sword":
      return 4;
    case "Gun":
      // Gun は例外あり。武器だけでは確定しない
      throw new Error("Gun pursuit hit_count requires exact skill data or exception mapping");
    case "Scythe":
      // Scythe は例外あり。武器だけでは確定しない
      throw new Error("Scythe pursuit hit_count requires exact skill data or exception mapping");
  }
}
```

### フォールバック込みの最終形

```ts
function getPursuitHitCountFallbackAware(ctx: {
  chara: string;
  style: string;
  weaponType: WeaponType;
  transformedPursuitSkillName?: string | null;
  skillsJson: Array<any>;
}): number {
  if (ctx.transformedPursuitSkillName === "ネコジェット・シャテキ") {
    return 5;
  }

  const pursuit = ctx.skillsJson.find(
    (s) =>
      s.name === "追撃" &&
      s.chara === ctx.chara &&
      s.style === ctx.style
  );
  if (pursuit) return pursuit.hit_count;

  switch (ctx.weaponType) {
    case "DoubleSword":
    case "LargeSword":
      return 2;
    case "Cannon":
    case "Shield":
    case "Claw":
      return 3;
    case "Sword":
      return 4;
    case "Gun":
      return getGunPursuitHitCount(ctx.chara);
    case "Scythe":
      return getScythePursuitHitCount(ctx.chara);
  }
}
```

### OD 増加量への使い方
- `pursuitHitCount = getPursuitHitCountFallbackAware(...)`
- その `pursuitHitCount` を既存の OD 増加量計算の hit 数入力に使う
- つまり「追撃の OD 増加量」は、追撃専用の別倍率を新設するのではなく、まず正しい `追撃ヒット数` を取得し、そのヒット数を通常の hit ベース OD 計算へ渡す、という扱いにしてください
