## ヘルプ記載内容

### 味方全体 ハイブースト (High Boost)

- SP消費量が2増加する
- スキル攻撃力が180%上昇する
- 攻撃力アップ(Attack Up)バフの効果が20%上昇する
- デバフ効果が20%上昇する
- 回復効果が50%上昇する
- ※ SP消費量増加効果は重複しない

### 味方全体 SP Limit Overwrite

- SP limit becomes 30 (SPの上限が30になる)
- ※ Default limit is 20 (※デフォルトの上限は20)

## シミュレーター実装情報

- **内部名**: `HighBoost` / `SpLimitOverwrite`

### 詳細仕様 (解析情報)

- 味方全体にかかる極めて強力なオーラ状態、または戦闘全体に及ぶフィールドルール変更等として機能します。
- スキル威力上昇（180%）、バフ増強（+20%）、デバフ増強（+20%）、回復増強（+50%）といった多様な恩恵をもたらします。
- 代償として、スキルのSP消費量が+2されるペナルティが発生します（このSP増加ペナルティは重複しません）。
- 同時に「SP上限の書き換え（SP Limit Overwrite）」が発生し、パーティ全体の最大SP所持上限が通常の20から**30**へと拡張されます。

### 現在の実装状態

本シミュレーターでは、`HighBoost` を味方側の永続 `statusEffects` として保持し、`SpLimitOverwrite` と併せて battle start passive から付与する。

現在反映している内容は次のとおり。

1. **SP消費判定**: `consumeType=Sp` かつ `spCost > 0` の skill は、`HighBoost` 中に消費SP `+2` で preview / commit される。増加は非重複。
2. **SP最大値(Cap)の上書き**: `SpLimitOverwrite` により SP 上限を `30` として扱う。
3. **スキル攻撃力補正**: `preview` / `record` の `specialPassiveModifiers.highBoostSkillAtkRate` に `1.8` を保持する。
4. **バフレート強化**: `AttackUp` 系バフ付与量を `1.2x` で適用する。
5. **デバフレート強化**: 敵デバフ付与量を `1.2x` で適用する。
6. **ヒール量強化**: `HealSp`、`HealDpRate`、再生系、受動回復を `1.5x` で適用する。
