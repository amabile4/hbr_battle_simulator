# ヘルプ：HEAVEN BURNS RED > バトル > ハイブースト

## 基本情報

- **URL**: (ユーザー提供データ/解析テキスト準拠)
- **カテゴリ**: HEAVEN BURNS RED > バトル

## ヘルプ記載内容

### 味方全体 ハイブースト (High Boost)

- SP消費量が2増加する
- スキル攻撃力が180%上昇する
- 攻撃力アップ(Attack Up)バフの効果が20%上昇する
- デバフ効果が20%上昇する
- 回復効果が50%上昇する
- ※ SP消費量増加効果は重複しない

### 味方全体 SP Limit Overwrite

- SP limit becomes 30 (SPの上限が30になる)
- ※ Default limit is 20 (※デフォルトの上限は20)

## シミュレーター実装情報

- **内部名**: `HighBoost` / `SpLimitOverwrite`

### 詳細仕様 (解析情報)

- 味方全体にかかる極めて強力なオーラ状態、または戦闘全体に及ぶフィールドルール変更等として機能します。
- スキル威力上昇（180%）、バフ増強（+20%）、デバフ増強（+20%）、回復増強（+50%）といった多様な恩恵をもたらします。
- 代償として、スキルのSP消費量が+2されるペナルティが発生します（このSP増加ペナルティは重複しません）。
- 同時に「SP上限の書き換え（SP Limit Overwrite）」が発生し、パーティ全体の最大SP所持上限が通常の20から**30**へと拡張されます。

### 実装メモ

本シミュレーター内では、味方全体（Party）に及ぼすステートとして、あるいは特定の個別バフとして `HighBoost` および上限拡張状態を保持管理する機構が必要です。

ダメージ計算およびスキル効果の適用フロー（Phase 等）において、以下の割り込み・補正ロジックを実装する必要があります。

1. **SP消費判定**: プレビュー（previewTurn）およびターンコミット時のスキルSP消費量計算で、自身にハイブーストの恩恵がある場合、計算上の要求SPを `+2` して判定・消費させます。
2. **SP最大値(Cap)の上書き**: ターンのSP回復フェーズや、DP/SP状態を管理する `CharacterStyle` モデルにおいて、`SpLimitOverwrite` が有効な間は `.sp.cap`（またはカンスト判定の上限値）を `20` から `30` に変更して運用します。
3. **ダメージ計算 (Skill ATK)**: 攻撃スキルによるダメージ計算時に、180%（1.8倍相当、または加算枠かは要検証）を威力枠に乗算します。
4. **バフレート強化**: エンハンスなどの Attack Up 系バフ付与時に、バフの倍率に `+20%` の補正をかけます。
5. **デバフレート強化**: 防御デバフなどの付与時に、デバフの倍率に `+20%` の補正をかけます。
6. **ヒール量強化**: DP回復やSP回復スキルの効果量計算時に、最終回復量に `+50%` の補正をかけます。
