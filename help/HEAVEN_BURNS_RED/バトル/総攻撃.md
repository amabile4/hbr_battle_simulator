# ヘルプ：HEAVEN BURNS RED > バトル > 総攻撃

## 基本情報

- **URL**: なし (ゲーム内共通仕様)
- **カテゴリ**: HEAVEN BURNS RED > バトル

## 概要 (ゲーム内仕様)

総攻撃は、全ての敵をブレイクした瞬間に発生する `HOLD UP!` から派生する特殊コマンドです。

- **発生条件**: 全ての敵をブレイクした瞬間に `HOLD UP!` が発生し、次の行動選択時に `総攻撃 (1回)` コマンドが出現する
- **発動可能条件**:
  - 部隊に `総攻撃` アビリティを持つキャラがいる
    - モナ
    - クイーン
    - ヴァイオレット (データ追加時に自動対応)
  - 生存しているすべての敵が `DownTurn` 中である
- **効果**:
  - コマンド選択中に `総攻撃` を選択して発動する
  - ターンを進行させない
  - 敵全体に一斉攻撃を行う
  - 攻撃演出はジョーカーで固定
  - 貫通クリティカル扱い
  - 敵全体の破壊率を固定で `100%` 上昇させる
  - ODゲージを `生存敵数 x 17.5%` 増加させる
- **ODゲージ増加の扱い**:
  - 敵1体につき `17.5%` 固定
  - 属性耐性の影響を受けない
  - ODピアスなどのアクセサリ補正を受けない
  - ODゲージ上昇量バフの影響を受けない
- **制限・消失条件**:
  - 味方の攻撃力アップなどのバフ効果は受けない
  - `総攻撃` アビリティ持ちを複数編成しても、出現数は重複しない
  - 総攻撃を発動すると実行権利は消失する
  - いずれかの敵のダウンターンが終了すると実行権利は消失する

## シミュレーター実装情報 (関連データや実装すべきトリガー・状態の定義)

- **内部状態案**: `turnState.holdUpActive`
- **関連 enemy status**: `DownTurn`
- **関連コマンド**: `総攻撃`
- **関連スタイル/キャラ**:
  - モナ (`roleabi.name === "総攻撃"`)
  - クイーン (`roleabi.name === "総攻撃"`)
  - ヴァイオレット (現データでは未定義、将来 `roleabi` 追加時に自動対応)
- **現状の実装ステータス**: 実装済み

実装済みの範囲:

- `turnState.holdUpActive` による `HOLD UP!` 権利管理
- 全ての生存敵が新たに `DownTurn` になったターン終了後の `holdUpActive = true`
- `DownTurn` 終了などで全生存敵ダウン状態でなくなった場合の `holdUpActive = false`
- `ActivateAllOutAttack` replay operation
- UI Next の `総攻撃` ボタンと operation chip
- 生存敵全体の破壊率 `+100%`
- ODゲージ `生存敵数 x 17.5%` 固定加算、通常上限 `300%` clamp
- session save/load、edit/recommit 時の operation 保持と再計算

実装状態:

```js
turnState.holdUpActive = true
```

実装判定:

1. 生存しているすべての敵が `DownTurn` 中か
2. パーティ内に `総攻撃` アビリティ持ちがいるか
3. `holdUpActive` が true か

実装済みの総攻撃コマンド効果:

```js
destructionRateByEnemy[targetEnemyIndex] += 100
turnState.odGauge += aliveEnemyCount * 17.5
turnState.holdUpActive = false
```

ODゲージは通常上限の `300%` でクランプします。

## シミュレーターでの解決 (Resolution) と評価

総攻撃は、通常のスキル action ではなく、ターンを消費しない特殊コマンドとして解決します。

### HOLD UP! 判定

敵を攻撃してブレイクしたフェーズ処理の結果、生存しているすべての敵が新たに `DownTurn` ステータスを付与された状態になった場合、`holdUpActive = true` とします。

### コマンド選択

行動選択フェーズでは、以下を満たす場合のみ `総攻撃` コマンドを表示します。

1. `turnState.holdUpActive === true`
2. パーティ内に `総攻撃` アビリティを持つキャラがいる
3. 生存しているすべての敵が `DownTurn` 中である

複数の `総攻撃` アビリティ持ちを編成していても、コマンドのストック数は増やしません。

### 実行処理

総攻撃を実行した場合、通常のターン進行処理は行いません。

スキップする処理:

- `turnCount` の増加
- バフ/デバフの残りターン消費
- ターン開始時SP回復
- ターン終了時SP回復
- PlayerTurnEnd / EnemyTurnEnd 系の通常経過処理

適用する処理:

1. 生存しているすべての敵の破壊率を `+100%`
2. ODゲージを `生存敵数 x 17.5%` 加算
3. ODゲージを最大値でクランプ
4. `holdUpActive = false`

総攻撃は貫通クリティカル扱いですが、通常スキル由来のダメージ・hit・OD計算とは分離します。味方バフ、属性耐性、ODピアス、OD上昇量バフは反映しません。

### 消失処理

敵のダウンターン経過処理で、いずれかの敵の `DownTurn` が終了して状態から外れた場合、`holdUpActive = false` に戻します。

また、敵の撃破や状態変化によって「生存しているすべての敵がダウン中」ではなくなった場合も、`holdUpActive = false` に戻します。
