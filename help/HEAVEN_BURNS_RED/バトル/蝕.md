# ヘルプ：HEAVEN BURNS RED > バトル > 蝕

## 基本情報
- **URL**: なし（ゲーム内説明画像ベース）
- **カテゴリ**: HEAVEN BURNS RED > バトル

## ヘルプ記載内容
蝕（しょく）

- 全能力が100減少
- 全能力を下げる効果を重ねがけした場合、効果値の高いほうで算出される
- 本効果を重ねがけした場合、効果ターンの長い方で上書きされる

## シミュレーター実装情報
- **内部名**: `Undermine`
- **対象**: enemy status (`enemyState.statuses`)
- **付与経路**: enemy debuff と同じ status 保存経路を使用し、`EnemyTurnEnd` などの duration をそのまま保持する
- **表示**: `ui-next` の敵状態アイコン / enemy detail popup / preview status popup で `蝕` ラベルと専用アイコンを表示する
- **runtime support**: `classifyEnemyStatusPartRuntimeSupport()` で supported 扱い
- **数値影響の扱い**:
  - 状態管理・表示・session/replay 再計算は `Undermine` として対応済み
  - 「全能力-100」を個別ステータス係数へ落とし込む実ダメージ計算は、現時点では専用数式を別実装していない
  - そのため現状のシミュレーターでは、まず enemy status としての観測・保持・UI 表示を正とする
- **重ねがけ**: 既存の enemy status 上書きルールに従い、同一種別 `Undermine` は長い残ターン側を採用する

### シミュレーターでの解決 (Resolution) と評価

`蝕` は実データ上 `skill_type: Undermine` として供給される新規 enemy debuff である。

本シミュレーターでは、まず shared engine / replay / `ui-next` の enemy status 基盤へ `Undermine` を接続し、以下を保証する。

1. スキル使用時に `Undermine` が対象 enemy に付与される
2. commit / recalculate / session save-load 後も `Undermine` が保持される
3. enemy detail popup と turn row の status icon で `蝕` として視認できる
4. source skill desc を含む preview 表示でも既存 enemy status と同じ扱いになる

数値面の「全能力-100」はヘルプ仕様として明記するが、現段階では専用の 6 能力パラメータ減算を damage formula へ直接注入していない。そのため、この変更の主眼は「実装済み status を未対応扱いにしない」「UI と文書を揃える」ことに置く。
