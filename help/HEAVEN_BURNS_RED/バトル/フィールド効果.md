## 追加調査メモ (2026-03-13)

- `ZoneUpEternal` は「フィールド性能+15%」と「ターン指定ありフィールドを永続化」の **二効果パッシブ** として扱う必要がある
  - どちらか片方の効果ではなく、同じ modifier から 2 つの結果が出る前提で実装整理する
- `passives.json` で確認できる `ZoneUpEternal` パーツは現状 3 件
  - `武運長久` x1
  - `天長地久` x2
- 現在確認できる `ZoneUpEternal` パーツの `power[0]` はすべて `0.15`
  - 実装上は `+0.15` 固定値にせず `part.power[0]` を読むのが安全
- `武運長久`
  - `timing: OnPlayerTurnStart`
  - `condition: MoraleLevel()>=6`
  - `target_type: Self`
  - `effect.exitCond: PlayerTurnEnd`, `exitVal: [1,0]`
  - 解釈: そのプレイヤーターン中だけ「自分が展開するフィールド」に二効果を付与する modifier
- `天長地久`
  - `timing: OnFirstBattleStart`
  - `condition: ""`
  - `target_type: Field`
  - `effect.exitCond: Eternal`
  - 解釈: バトル全体を通して、自身のフィールド展開に二効果を与える常在 modifier
- フィールド本体の observed power は現状 `1.35` / `1.65` / `1.8`
  - `ZoneUpEternal(+0.15)` を乗せると `1.50` / `1.80` / `1.95`
- 実装上の重要点
  - **性能上昇** と **永続化** は別々に解決する
  - 既に永続のフィールドには「永続化」は追加効果を生まないが、「性能+15%」はそのまま適用される
  - `part.effect` は `ZoneUpEternal` modifier 自体の有効期間を表しており、フィールド本体の duration と直接同一視しない
  - 2026-03-13 実装反映: `part.power[0]` ベースで `powerRate` を加算し、`remainingTurns !== null` のフィールドだけ永続化する
