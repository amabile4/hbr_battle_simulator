# スキル攻撃ダメージ計算式 仕様書

## 1. 計算式表現（数式定義）

最終ダメージ = ベース値 * 基礎_属性バフ枠 * 敵のデバフ枠 * 連撃バフ枠 * トークンバフ枠 * クリティカル補正枠 * 武器相性枠 * 属性相性枠 * 破壊率_特効_ランダム枠

各枠の計算式は以下の通り定義する。

[ベース値]
ベース値 = スキル攻撃力

[基礎_属性バフ枠]
基礎_属性バフ枠 = 1.0 + スキル攻撃力バフ + スキル属性攻撃力バフ + チャージ + アクセサリ + パッシブスキル他 + 心眼バフ + 属性フィールド

[敵のデバフ枠]
敵のデバフ枠 = 1.0 + 防御ダウン + 永続防御ダウン + 属性ダウン + 永続属性ダウン + DP防御ダウン + パッシブスキル他 + 脆弱 + 永続脆弱

[連撃バフ枠]
連撃バフ枠 = 1.0 + 連撃バフ + パッシブスキル他

[トークンバフ枠]
トークンバフ枠 = 1.0 + トークンバフ

[クリティカル補正枠]
・クリティカル発生時
クリティカル補正枠 = 1.5 + クリ威力 + 属性クリ威力 + パッシブスキル他
・非クリティカル発生時
クリティカル補正枠 = 1.0

[武器相性枠]
武器相性枠 = 敵の武器属性相性倍率（斬・突・打）

[属性相性枠]
属性相性枠 = 1.0 + 属性相性値 + 属性ダウン

[破壊率_特効_ランダム枠]
破壊率_特効_ランダム枠 = 現在の破壊率 * (1.0 + 特効倍率) * 乱数

## 2. 各項目の補足説明

### ■ ベース値
* **スキル攻撃力**: キャラクターのステータスおよびスキル固有の威力から算出される基礎値。※クリティカル時は敵能力値を「-50」として計算する仕様が別途存在する。

### ■ 基礎・属性バフ枠
* **スキル攻撃力バフ / スキル属性攻撃力バフ**: スキル効果によって上昇する各種攻撃バフ値。
* **チャージ**: チャージスキルの効果値。
* **アクセサリ**: 装備している各種リングやピアスの効果値。
* **パッシブスキル他**: アビリティやスタイルパッシブなどの効果値。
* **心眼バフ**: 心眼によって付与されるバフ値。
* **属性フィールド**: 属性強化フィールド（弱:0.35 / 通常:0.65 / 大:0.80 など）の効果値。

### ■ 敵のデバフ枠
* **防御ダウン / 永続防御ダウン**: 敵に付与されている防御デバフ値。
* **属性ダウン / 永続属性ダウン**: 敵に付与されている属性防御デバフ値。
* **DP防御ダウン**: 敵のDPに対する防御デバフ値。
* **パッシブスキル他**: 敵の弱体化に関わるパッシブ効果値。
* **脆弱 / 永続脆弱**: 敵に付与されている脆弱の効果値。

### ■ 連撃バフ枠
* **連撃バフ**: 連撃数（連撃数上昇バフ）による一撃あたりの重み付け補正値。

### ■ トークンバフ枠
* **トークンバフ**: 特定のスタイルが持つ「トークン」の蓄積数に応じた倍率補正値。

### ■ クリティカル補正枠
* **クリ威力 / 属性クリ威力**: クリティカル時の威力を増加させるバフ・スキル効果値。

### ■ 武器相性枠
* **敵の武器属性相性倍率**: 敵の弱点・耐性（等倍:1.0 / 弱点:1.5 / 耐性:0.5 など）。

### ■ 属性相性枠
* **属性相性値**: 元素属性（火・水・雷・氷・光・闇）の弱点・耐性値。
* **属性ダウン**: エレメンタルダウン等、デバフとは別枠で加算される属性ダウン値。

### ■ 破壊率＆特効＆ランダム枠
* **現在の破壊率**: 敵の現時点の破壊率（100%〜999%を内部的に 1.0〜9.99 として扱う）。
* **特効倍率**: スキルが持つ「HP+50%」などの特定状況下の特効効果値（DP攻撃時はDP特効、HP攻撃時はHP特効を適用）。
* **乱数**: 0.9 〜 1.1 の間でランダムに変動する最終ブレ幅。

## 3. データ定義仕様

ゲーム内およびシミュレーター内部のJSONデータ（`skills.json`, `passives.json` など）で定義されているバフ・デバフの `skill_type` やパラメータと、ダメージ計算式の各枠との対応関係は以下の通りです。

### 1. 基礎・属性バフ枠
計算式：`1.0 + スキル攻撃力バフ + スキル属性攻撃力バフ + チャージ + アクセサリ + パッシブスキル他 + 心眼バフ + 属性フィールド`

| 枠内の項目 | 内部データ定義（`parts[].skill_type` / 条件） | 効果値 `power`（基本値）の範囲 | 日本語表示名 (`STATUS_LABELS`) | 補足説明・適用仕様 |
| :--- | :--- | :--- | :--- | :--- |
| **スキル攻撃力バフ** | `AttackUp`<br>（`elements` が空配列かつ、対象がスキル） | `0.05 〜 3.0`<br>(`+5% 〜 +300%`) | 攻撃力アップ | 通常のスキル攻撃力を上昇させるバフ。通常発動（2回合算）か、単独発動（1つのみ採用）に分かれます（詳細は「4. バフの適用形式」を参照）。 |
| **スキル属性攻撃力バフ** | `AttackUp`<br>（`elements` に属性元素が含まれる場合） | `0.05 〜 3.0`<br>(`+5% 〜 +300%`) | 属性攻撃力アップ<br>（例: 火攻撃力アップ 等） | 特定の属性（元素属性または武器属性）を伴う攻撃の時のみ適用される攻撃力バフ。通常発動・単独発動が存在します。 |
| **チャージ** | `BuffCharge` | `0.2 〜 0.3`<br>(`+20% 〜 +30%`) | チャージ | スキル使用時に消費される、次のスキル威力を上昇させるバフ。ダメージ計算の基礎・属性バフ枠に加算されます。 |
| **パッシブスキル他** | `AttackUp` / `AttackUpPerToken`<br>（パッシブスキル・アビリティ由来） | `0.05 〜 0.2`<br>(`+5% 〜 +20%`) | 攻撃力アップ<br>（アビリティ/スタイルパッシブ） | キャラクターの限界突破アビリティやスタイルパッシブ（例：前衛にいるとき攻撃力+10%など）によるバフ値。 |
| **心眼バフ** | `MindEye` | `0.42 〜 1.2`<br>(`+42% 〜 +120%`) | 心眼 | 敵の弱点属性を突いた時のダメージを上昇させるバフ。単独発動タイプが多くを占めます。 |
| **属性フィールド** | `Zone` | `0.35` / `0.65` / `0.80`<br>(`+35%` / `+65%` / `+80%`) | フィールド（例: 火属性フィールド 等） | バトルフィールド全体に展開される属性強化フィールド。展開されている属性フィールドと攻撃属性が一致する場合に適用されます（弱: 0.35 / 通常: 0.65 / 大: 0.80など）。 |

---

### 2. 敵のデバフ枠
計算式：`1.0 + 防御ダウン + 永続防御ダウン + 属性ダウン + 永続属性ダウン + DP防御ダウン + パッシブスキル他 + 脆弱 + 永続脆弱`

| 枠内の項目 | 内部データ定義（`parts[].skill_type` / 条件） | 効果値 `power`（基本値）の範囲 | 日本語表示名 (`STATUS_LABELS`) | 補足説明・適用仕様 |
| :--- | :--- | :--- | :--- | :--- |
| **防御ダウン / 永続防御ダウン** | `DefenseDown`<br>（`elements` が空配列かつ、対象が敵全体/単体） | `0.1 〜 1.0`<br>(`+10% 〜 +100%`) | 防御力ダウン | 敵の防御力を下げるデバフ。ターン制限のあるものと、戦闘中永続するもの（永続デバフ）が存在します。 |
| **属性ダウン / 永続属性ダウン** | `DefenseDown`<br>（`elements` に属性元素 `Fire`, `Ice` などが含まれる場合） | `0.1 〜 1.0`<br>(`+10% 〜 +100%`) | 属性防御力ダウン<br>（例: 火防御力ダウン 等） | 敵の特定属性に対する防御力を下げるデバフ。攻撃属性と一致する場合にのみ適用されます。 |
| **脆弱 / 永続脆弱** | `Fragile` | `0.3 〜 0.42`<br>(`+30% 〜 +42%`) | 脆弱 | 敵が弱点属性で受けるダメージを増幅するデバフ。弱点属性攻撃時のみ適用されます。 |

---

### 3. 連撃バフ枠
計算式：`1.0 + 連撃バフ + パッシブスキル他`

| 枠内の項目 | 内部データ定義（`parts[].skill_type`） | 効果値 `power`（基本値）の範囲 | 日本語表示名 (`STATUS_LABELS`) | 補足説明・適用仕様 |
| :--- | :--- | :--- | :--- | :--- |
| **連撃バフ** | `Funnel` | `2 〜 5`<br>(連撃数 2回 〜 5回) | 連撃数アップ | 連撃数上昇バフ。通常発動（最大2回分合算）と単独発動（1つのみ採用）に分かれます。 |

---

### 4. トークンバフ枠
計算式：`1.0 + トークンバフ`

| 枠内の項目 | 内部データ定義（`parts[].skill_type`） | 効果値 `power`の範囲 | 日本語表示名 (`STATUS_LABELS`) | 補足説明・適用仕様 |
| :--- | :--- | :--- | :--- | :--- |
| **トークンバフ** | `TokenSet` で獲得したトークンを利用した倍率補正 | `1.0 〜 10.0`<br>(トークン所持数) | トークン上昇 / トークン | キャラクターが蓄積したトークン（最大10個）の数に応じて、対応する特効スキル（`TokenAttack`）などのダメージ計算時に適用される倍率補正。 |

---

### 5. クリティカル補正枠
計算式：
* クリティカル発生時：`1.5 + クリ威力 + 属性クリ威力 + パッシブスキル他`

| 枠内の項目 | 内部データ定義（`parts[].skill_type` / 条件） | 効果値 `power`（基本値）の範囲 | 日本語表示名 (`STATUS_LABELS`) | 補足説明・適用仕様 |
| :--- | :--- | :--- | :--- | :--- |
| **クリ威力** | `CriticalDamageUp`<br>（`elements` が空配列） | `0.08 〜 1.3`<br>(`+8% 〜 +130%`) | クリティカルダメージアップ | クリティカル発生時のダメージ倍率を底上げするバフ。通常発動（最大2回分合算）と単独発動（1つのみ採用）に分かれます。 |
| **属性クリ威力** | `CriticalDamageUp`<br>（`elements` に属性元素が含まれる場合） | `0.08 〜 1.3`<br>(`+8% 〜 +130%`) | 属性クリティカルダメージアップ | 特定の属性でクリティカルが発生した時のみ適用されるクリティカル威力バフ。 |

## 4. バフの適用形式（通常発動と単独発動）の分類

バトル中のダメージ計算において、バフ効果（スキル攻撃力バフ、属性攻撃力バフ、連撃バフ、クリティカルダメージアップなど）は、スキルの適用形式（通常発動と単独発動）に応じて解決ロジックが異なります。シミュレーター内部およびゲーム仕様との突き合わせは以下の通りです。

### ■ 通常発動（通常バフ / 回数制限バフ）
* **スキル説明での表現**: 「スキル攻撃力を上げる」などの一般的なバフ表記（特に回数が指定されるか、ターン数のみの永続でないもの）。
* **データ定義 (`skills.json`)**: `parts[].effect.limitType` が `Default` 等で、`exitCond` が `Count`（回数制限）に設定されているもの。
* **ゲーム内＆シミュレーター実装挙動**:
  * 同種バフ（例：スキル攻撃力アップ）が複数付与されている場合、効果値（`power`）が高いものから**最大2つまで合算**されます（2重の重ねがけ合算が可能）。
  * 適用時には回数（`remaining`）が消費されます。

### ■ 単独発動（専用限定バフ）
* **スキル説明での表現**: スキル説明文の中に **`[単独発動]`** の表記があるもの。
  * 例：「3ターンの間 自身のスキル攻撃力を著しく上げ[単独発動]」など。
* **データ定義 (`skills.json`)**: `parts[].effect.limitType` が **`Only`**（専用スキル限定バフ）に設定されているもの。
* **ゲーム内＆シミュレーター実装挙動**:
  * 複数付与されていても、**最も効果が高い1つの効果値のみ**が採用され、重複して合算されることはありません。
  * 通常、ターン経過（`exitCond` が `PlayerTurnEnd` など）で管理されます。

### ■ シミュレーターでの解決 (Resolution) と評価ルール
シミュレーターのバフ解決関数（[turn-controller.js](file:///Users/ram4/git/hbr_battle_simulator/src/turn/turn-controller.js#L7114-L7150)）では、現在付与されているバフから通常発動（Count）と単独発動（Only）を抽出し、以下のルールで競合解決を行います。

1. **通常発動枠（合算）と単独発動枠（単発）の比較**:
   * 通常発動（Countバフ）の上位2つの合算値と、単独発動（Onlyバフ）の最大値のうち、**より効果値が高い方のバフグループが優先採用**されます。
   * 計算式：`adopted = 通常発動の合算値(最大2枚) >= 単独発動の最大値(1枚) ? 通常発動を採用 : 単独発動を採用`
2. **永続バフとの合算**:
   * 上記で採用されたバフグループの効果値は、限界突破アビリティなどの「永続バフ」および「パッシブスキル他」とさらに合算されます。
3. **スキルとパッシブの共存**:
   * 「アクティブスキルによる単独発動バフ」と「パッシブスキルによるバフ」は別枠であり、それぞれ独立して効果を発揮します。
