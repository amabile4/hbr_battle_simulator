# skill: generate_help_manual

このスキルは、特定のゲーム仕様・ステータスに関する `skills.json` 内の該当スキル一覧を抽出し、プロジェクト標準フォーマットに従ったヘルプドキュメントを自動生成するためのルール群とスクリプトセットです。

## 使用条件 (When to use)
- ユーザーから「あるスキル効果・ステータス効果のヘルプやマニュアルを作って」と依頼された場合。
- ゲーム内テキスト（例:「SPが0以上であれば使用可能」や「速弾き」など）を持つ全てのアクティブなスキルをリストアップしたい場合。

## 手順 (Workflow)

1. **スクリプトの実行**
   提供されているPythonスクリプトを使用して、抽出とベースとなるMarkdownファイル生成を行います。
   このスクリプトは `characters.json` と照らし合わせ、キャラクター名から英語やローマ字名を除去した「日本語名称のみ」を抽出・適用します。

   実行コマンド例:
   ```bash
   python3 .agents/skills/generate_help_manual/scripts/generate.py "検索したい語句(例: 速弾き)" "help/HEAVEN_BURNS_RED/バトル/出力ファイル名.md"
   ```

2. **手動追記・解説の入力**
   スクリプトが生成したファイルを開き、以下のセクションに具体的な仕様やシミュレーター実装上の注意点（Resolution）を書き加えます。
   - `## 概要`: ゲーム内での効果、挙動について解説する。
   - `**シミュレーターでの解決 (Resolution) と評価**`: シミュレーター内でどのようにこの効果を再現・管理するかの設計仕様を記述する。（既存の `速弾き.md` などを参照のこと）

3. **自動更新スキップリスト(SKIP_LIST)への登録**
   生成した `.md` ファイルが後続の自動更新スクリプトによって上書きされないように、`help/scripts/populate_templates.py` の `SKIP_LIST` に、手動作成したファイル名（拡張子なし）を追加してください。

   例: `SKIP_LIST = [..., "速弾き", "SPが0以上であれば使用可能"]`

4. **Gitへのコミット**
   新規作成したマニュアルと、更新した `populate_templates.py` をGitにコミットします。
